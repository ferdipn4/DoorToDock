Database Schema

DockSense -- Supabase PostgreSQL Entity-Relationship Diagram

bike_availability ~315,000 rows
id PK SERIAL Auto-increment primary key
timestamp TIMESTAMPTZ UTC observation time
station_id FK TEXT e.g. "BikePoints_809"
station_name TEXT Human-readable station name
available_bikes INTEGER Total bikes (standard + e-bikes)
standard_bikes INTEGER Standard pedal bikes
ebikes INTEGER Electric bikes
empty_docks INTEGER Free docking slots (prediction target)
total_docks INTEGER Station capacity
latitude FLOAT Station latitude
longitude FLOAT Station longitude
weather_data ~19,000 rows
id PK SERIAL Auto-increment
timestamp TIMESTAMPTZ UTC observation time
temperature FLOAT Degrees Celsius
humidity FLOAT Relative humidity %
precipitation FLOAT Rainfall mm/h
wind_speed FLOAT Wind speed m/s
weather_code INTEGER WMO weather code
description TEXT e.g. "light rain"
monitored_stations 21 rows
station_id PK TEXT e.g. "BikePoints_809"
station_name TEXT Display name
latitude FLOAT Station latitude
longitude FLOAT Station longitude
distance_m FLOAT Haversine distance to IC
walking_distance_m FLOAT Walking distance
walking_duration_s FLOAT Walking time (seconds)
temperature_readings ~19,000 rows
created_at TIMESTAMPTZ Auto-set by Supabase
temperature_c FLOAT KY-028 sensor reading
sensor_events ~20 rows
id PK SERIAL Auto-increment
timestamp TIMESTAMPTZ Event time (UTC)
event_type TEXT e.g. "departure"
confidence FLOAT Post-debounce confidence
Relationships
bike_availability .station_id ---> monitored_stations .station_id (many-to-one)
bike_availability .timestamp - - > weather_data .timestamp (join key for merged dataset, rounded to minute)
