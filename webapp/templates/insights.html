Data quality note

On average, 2.7 docks per station are blocked (total_docks does not equal empty_docks + available_bikes), likely due to maintenance. This is constant throughout the day. empty_docks and available_bikes are used directly from the API, never derived from each other.

The Nowcast model achieves high accuracy because the current dock count is the strongest predictor of the count 15 minutes later. Chapter 6

Design trade-offs

Every system involves trade-offs. Here are the key decisions we made and why.

Architecture decisions
Polling: 1/min vs 5/min
Chose 1/min
TfL updates every ~5 min but exact timing is unknown. Higher frequency catches changes promptly at the cost of 5x more storage.
Two models vs one
Two separate models
Nowcast (with lag feature) gets 3x better accuracy for live use. Forecast (no lag) enables planning but with higher error.
Baseline vs ML for Forecast
Historical Average wins
With only 22 days of data and limited weather diversity, per-(station, hour, weekday) averages outperform Random Forest and Gradient Boosting.
Technology choices
Open-Meteo vs OpenWeatherMap
Open-Meteo
No API key, unlimited calls, same data quality. OpenWeatherMap has a 1000 calls/day limit requiring key management.
Supabase vs MongoDB
Supabase PostgreSQL
SQL joins for time-series merging, familiar query language. MongoDB's flexible schema is unnecessary; joins are essential for our pipeline.
Heroku vs Raspberry Pi
Heroku
Reliable uptime, no hardware maintenance, student pack credits. Pi is cheaper long-term but requires physical setup and network configuration.
Sensor for training only
Validated, then retired
The KY-028 temperature sensor validated the weather API (r = 0.96) and enriched training data. After validation, the deployed model uses only the weather API for reliability.
Limitations
Rain analysis limitation
22 days of data
22 days of data with minimal heavy precipitation limits rain-effect analysis. Longer collection periods would enable seasonal and weather-pattern insights.
Nowcast model honesty
98.6% lag feature
The Nowcast model's accuracy is dominated by current dock count (98.6% importance). This is expected for short-horizon time-series prediction but means the model adds limited value beyond "check current availability." Its value is in the 15-minute forward projection and confidence scoring.
